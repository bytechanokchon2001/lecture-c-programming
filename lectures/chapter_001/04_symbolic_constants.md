# Symbolic Constants
Symbolic Constants (ค่าคงที่เชิงสัญลักษณ์) คือ **การตั้งชื่อหรือสัญลักษณ์ (Identifier)** ให้แทนค่าข้อมูลค่าหนึ่งที่คงที่ ซึ่งค่านี้จะ*ไม่สามารถเปลี่ยนแปลงได้ตลอดการทำงานของโปรแกรม* 

เป็นการแก้ปัญหาที่เกิดจาก **ตัวเลขปริศนา (Magic Number)** ในโค้ด

## ปัญหาของตัวเลขปรินา
จากตัวอย่างในหัวข้อก่อนหน้า การใส่ตัวเลขลงไปในโค้ดโดยตรง เช่น 300, 20 หรือ 0 เรียกว่า "Magic Numbers" ซึ่งมีข้อเสียคือ
1. **อ่านเข้าใจยาก** คนที่มาอ่านทีหลังอาจไม่รู้ว่า 300 คือค่าอะไร (เช่น ค่าสูงสุด หรือค่าเฉลี่ย ?)
2. **แก้ไขยาก** ถ้าเลข 300 ปรากฏอยู่ในโปรแกรมหลายจุด แล้วเราต้องการเปลี่ยนเป็น 500 เราต้องไล่แก้ทุกจุด ซึ่งเสี่ยงต่อการผิดพลาด

## การใช้ `#define`
K&R แนะนำให้ใช้ Symbolic Constants โดยใช้คำสั่ง `#define` ไว้ที่ตอนต้นของโปรแกรม รูปแบบคือ `#define (name) (value)`

    #define LOWER 0;    // อุณหภูมิต่ำสุด
    #define UPPER 300;  // อุณหภูมิสูงสุด
    #define STEP 20;    // จำนวนอุณหภูมิที่เพิ่มต่อการวนซ้ำ

## หลักการทำงาน
- เมื่อเรา **Compile** โปรแกรม ตัว **Pre-processor** จะนำชื่อที่เราตั้งไว้ (เช่น LOWER) ไปแทนที่ด้วยข้อความหรือตัวเลขที่กำหนดไว้ (เช่น 0) *ทั่วทั้งโปรแกรมก่อนที่จะเริ่มการแปลภาษาจริง*
- **ข้อสังเกต** หลังคำสั่ง #define จะไม่มีเครื่องหมาย semicolon (`;`) ปิดท้าย

## กฏการตั้งชื่อ
โดยธรรมเนียมปฏิบัติของภาษา C (Convention) ชื่อของ Symbolic Constant มักจะใช้ **ตัวพิมพ์ใหญ่ทั้งหมด (UPPERCASE)** เพื่อให้*แยกแยะได้ชัดเจนจากตัวแปรทั่ว*ไปที่เป็นตัวพิมพ์เล็ก

## ประโยชน์ที่จะได้รับ
- **Readability (การอ่านง่าย)** โค้ดจะอ่านแล้วเข้าใจทันที เช่น `for (fahr = LOWER; fahr <= UPPER; fahr = fahr + STEP)`
- **Maintainability (การดูแลรักษา)** หากต้องการเปลี่ยนขีดจำกัดของโปรแกรม เราแก้เพียงจุดเดียวที่ด้านบนของไฟล์ ค่าก็จะเปลี่ยนไปทั้งโปรแกรมทันที
