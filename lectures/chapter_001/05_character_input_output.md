# Character Input and Output
การจัดการข้อมูลพื้นฐานที่สุดของภาษา C คือการมองข้อมูลทุกอย่าง (ไม่ว่าจะเป็นข้อความจากคีย์บอร์ดหรือไฟล์) เป็น **stream of characters** หรือ **ลำดับของตัวอักษรต่อเนื่องกัน**

### ฟังก์ชันพื้นฐาน `getchar` และ `putchar`
C ใช้ฟังก์ชันที่เรียบง่ายที่สุดสองตัวในการรับส่งข้อมูล โดยแบ่งเป็นดังนี้
- `getchar()` อ่านตัวอักษร ทีละตัว จากอินพุตมาตรฐาน (มักจะเป็นคีย์บอร์ด)
- `putchar(c)` แสดงผลตัวอักษรที่อยู่ในตัวแปร c ออกมาทีละตัว

### การจัดการกับ "End of File" (EOF)
ปัญหาคือโปรแกรมจะรู้ได้อย่างไร **ว่าข้อมูลหมดแล้ว ?**
- C ใช้ค่าพิเศษที่เรียกว่า `EOF` (End of File) ซึ่งเป็นค่าคงที่ที่ถูกนิยามไว้ใน `<stdio.h>`
- เมื่อ `getchar()` อ่านจนไม่เหลือข้อมูลแล้ว **มันจะคืนค่า EOF ออกมา**

**จุดสำคัญ** เราต้องใช้ตัวแปรชนิด int (ไม่ใช่ char) มารับค่าจาก `getchar()` <mark>เพราะค่า EOF มีขนาดใหญ่เกินกว่าที่ตัวแปร char บางประเภทจะเก็บได้</mark>

## File Copying
ตัวอย่างโปรแกรมที่รับค่ามาแล้วพ่นออกไปทันที (Copy input to output) ซึ่งมีประโยคที่กลายเป็นเอกลักษณ์ของ C *คือการใช้ Assignment ในเงื่อนไขของ while*

    #include <stdio.h>

    int main()
    {
        int c;

        // อ่านค่า เก็บเข้า c แล้วเช็คทันทีว่าใช่ EOF ไหม
        while ((c = getchar()) != EOF)
        {
            putchar(c);
        }

        return 0;
    }

**ข้อควรระวัง** ต้องใส่เครื่องหมายวงเล็บล้อมรอบ (c = getchar()) เพราะตัวดำเนินการ != มีลำดับความสำคัญ (Precedence) สูงกว่า = ถ้าไม่ใส่ วงเล็บ c จะได้รับค่า 0 หรือ 1 (ผลจากการเปรียบเทียบ) แทนที่จะได้รับตัวอักษรจริงๆ

## Character Counting
ผู้เขียนใช้พื้นฐานการรับส่งข้อมูลนี้มาสร้างโปรแกรมเล็กๆ อีก 3 ตัว
1. **Character Counting**: การนับจำนวนตัวอักษรทั้งหมด (รวมเว้นวรรคและขึ้นบรรทัดใหม่)
2. **Line Counting**: การนับบรรทัด โดยเช็คจากตัวอักษรพิเศษ \n (Newline)
3. **Word Counting**: การนับคำ ซึ่งมีความซับซ้อนขึ้นอีกนิด โดยต้องใช้ตัวแปร "State" เพื่อเช็คว่าตอนนี้เรากำลัง "อยู่ในคำ" หรือ "อยู่นอกคำ"
